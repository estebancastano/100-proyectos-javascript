# 🚀 100 Proyectos de JavaScript  

¡Bienvenido a **100 Proyectos de JavaScript**! 🎉 Este repositorio es una colección colaborativa de proyectos diseñados para aprender, practicar y explorar todo el potencial de JavaScript.  

Nuestra misión es construir una comunidad donde los desarrolladores, desde principiantes hasta expertos, puedan aprender juntos y contribuir con sus propias ideas.  

---

## 🌟 Objetivos del proyecto  
- Crear una colección de 100 proyectos en JavaScript, desde básicos hasta avanzados.  
- Ayudar a los desarrolladores a mejorar sus habilidades de codificación y colaborar en proyectos de código abierto.  
- Inspirar a otros a aprender JavaScript a través de ejemplos prácticos.

## 🤝 Cómo colaborar  

### 1. Haz un fork del repositorio  
Haz clic en el botón `Fork` en la parte superior de esta página para crear una copia del repositorio en tu cuenta.  

### 2. Clona tu fork en tu máquina local  
```bash
git clone https://github.com/tu-usuario/100-proyectos-de-javascript.git
```
### 3. Crea una rama para tu proyecto o contribución
```bash
git checkout -b nombre-de-tu-rama  
```
### 4. Añade tu proyecto o realiza mejoras
- Crea una nueva carpeta dentro de src/pages/proyectos/ con el nombre de tu proyecto (e.g., proyecto-03).
Añade tus archivos necesarios (index.astro, script.js, estilos, etc.).
Asegúrate de que tu proyecto sea funcional y esté bien documentado.
### 5. Realiza un commit y sube los cambios
```bash
git add .  
git commit -m "Añadido [nombre del proyecto]"  
git push origin nombre-de-tu-rama  
```
### Sobre los commits:
### 📋 Estándares de Código

#### Commits

Si es posible describe tus proyectos para que los mantenedores los puedan analizar de una forma más rápida y eficiente.

- `feat:` - Nuevas características
- `fix:` - Correcciones de bugs
- `docs:` - Cambios en documentación
- `style:` - Cambios que no afectan el código (espacios, formato, etc)
- `refactor:` - Refactorización del código
- `test:` - Añadir o modificar tests
- `chore:` - Cambios en el proceso de build o herramientas auxiliares

Ejemplo: `feat: add newsletter subscription component`
### 6. Abre un Pull Request
- Ve a la página del repositorio original en GitHub.
- Haz clic en New Pull Request.
- Describe brevemente tu contribución.
### 7. ¡Espera la revisión! 🎉
